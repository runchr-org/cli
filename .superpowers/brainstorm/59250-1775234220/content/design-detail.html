Design Decisions

  • 4 groups: SUMMARY (what + metrics), CODE (files + tools + decisions), LEARNINGS (kept separate — different from summary findings), SIGNALS (all facets with evidence)
  • Sub-section headers: Dim uppercase labels inside each box — visually distinct from box titles
  • Evidence rendering: Indented with ↳ prefix in dim style, directly below parent item
  • Scores inline: Overall score in amber, sub-scores dim on same line
  • Tool usage: Compact single-line with dot separators and counts in parens
  • Empty sections omitted: If a session has no failure loops, that sub-section header doesn't appear. If an entire box would be empty, the box is omitted.
  • Metadata header unchanged: SESSION + TOKENS boxes stay as-is (fixed, not scrolled)
