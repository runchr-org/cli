Detail Pane Layout Approaches

How to render 17 sections of session data in a scrollable detail pane

A

Separate Boxes Per Section

Each of the 17 sections gets its own bordered box with a title header, matching the current style (SUMMARY, FRICTION, LEARNINGS boxes). Empty sections are omitted entirely.

Pros

  • Consistent with current design
  • Clear visual boundaries
  • Easy to scan section titles

Cons

  • Borders add vertical bulk — more scrolling
  • 17 boxes may feel heavy
B

Compact Grouped Layout

Group related sections under shared headers. Scores & Stats become a dense 2-line block at top. Code context (files, tools, rationale, tradeoffs, patterns) share one box. Insights (all facets) share one box with labeled sub-sections. Only the top-level groups get bordered boxes.

C

Hybrid: Dense Header + Separate Detail Boxes

Merge scores and stats into the existing metadata header (always visible, not scrolled). Keep the scrollable area for content-heavy sections, each in its own box. Best of both: key stats are always visible, detail sections stay clean and scannable.
